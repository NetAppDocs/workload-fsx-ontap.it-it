= Gestisci automaticamente la capacità del file system e gli inode in Workload Factory
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Abilitando la gestione della capacità, NetApp Workload Factory aggiunge automaticamente storage incrementale a un file system FSx for ONTAP man mano che le esigenze di capacità cambiano nel tempo. Inoltre, abilitare questa funzionalità elimina la necessità di monitorare manualmente la capacità.



== A proposito di questa attività

Il sistema esegue la scansione del file system FSx for ONTAP ogni 30 minuti. Verifica se è necessario aggiungere storage incrementale e controlla gli inode di volume disponibili. Il conteggio aumenta in base alle soglie di gestione automatica della capacità.

Abilitare la gestione degli inode consente a NetApp Workload Factory di aggiungere automaticamente inode incrementali a un file system FSx for ONTAP man mano che le esigenze di capacità cambiano. Questa funzionalità elimina la necessità di monitorare manualmente gli inode.

Questa funzione può essere gestita da un solo account.

La quantità massima di capacità di storage SSD per tutti i file system FSX per ONTAP è di 524.288 GiB. Per richiedere un aumento delle quote, fai riferimento alla link:https://docs.aws.amazon.com/fsx/latest/ONTAPGuide/limits.html["Quote"^] documentazione di AWS FSX per NetApp ONTAP.



== Consentire la gestione automatica della capacità

Abilita la gestione automatica della capacità per aggiungere automaticamente spazio di archiviazione incrementale fino al limite massimo di dimensione per un file system FSx per ONTAP .

.Prima di iniziare
Prima di iniziare, prendere in considerazione quanto segue:

* Devilink:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["concedere credenziali con la politica di autorizzazione _visualizzazione, pianificazione e analisi_"^] in Workload Factory per completare questa attività.
* Per assicurarsi che il volume degli inode aumenti insieme alla capacità di archiviazione, è necessario associare un collegamento. link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["Scopri come associare un collegamento esistente o come creare e associare un nuovo collegamento"]. Dopo l'associazione del collegamento, tornare a questa operazione.
* Si consiglia di non abilitare questa funzionalità durante la migrazione dei dati, poiché AWS impone un periodo di attesa minimo di sei ore tra un aumento della capacità dell'SSD e l'altro. Questa restrizione potrebbe ritardare gli adeguamenti, quindi è opportuno pianificare di conseguenza.


.Fasi
. Accedere utilizzando uno dei link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["esperienze di console"^].
. Seleziona il menu image:workloads-menu-icon.png["L'icona del menu hamburger viene utilizzata per navigare tra carichi di lavoro quali storage, eda, ai, database, vmware e amministrazione."] e quindi seleziona *Archiviazione*.
. Dal menu Archiviazione, selezionare *FSx per ONTAP*.
. Da *FSx for ONTAP*, selezionare il menu delle azioni del file system per abilitare la gestione automatica della capacità.
. Selezionare *Gestisci*.
. In Informazioni, seleziona l'icona a forma di matita accanto a *capacity management*. L'icona a forma di matita appare accanto alla freccia a discesa quando passi il mouse sulla riga *capacity management*.
. Nella finestra di dialogo *gestione della capacità*, fornire quanto segue:
+
.. *Credenziali*: Selezionare le credenziali con autorizzazioni _automatizza_ dal menu a discesa.
.. Selezionare il pulsante di abilitazione per *abilitare la gestione automatica della capacità*.
+
In alternativa, disattivare la funzione. Se è necessario aumentare la capacità del file system, è necessario prima disabilitare la gestione automatica della capacità.

.. *Modalità*: Seleziona la modalità per gestire automaticamente la capacità. Le opzioni sono modalità incrementale e adattiva.
+
*** *Modalità incrementale*: aumenta la capacità di una quantità fissa alle soglie.
+
**** *Soglia di avviso*: imposta la soglia di avviso su un valore inferiore all'aumento della soglia per attivare una notifica dal servizio di notifica Workload Factory.  Il valore predefinito è 70%.
+
L'impostazione della soglia di avviso è disponibile solo selink:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["abilitato il servizio di notifica Workload Factory"^] .

**** *Aumento soglia*: immettere l'aumento percentuale massimo per il file system FSx for ONTAP .  Il valore predefinito è 80%.
+
Questa è la soglia alla quale Workload Factory attiva un processo per aumentare la capacità. Ad esempio, se il file system raggiunge l'80% della capacità, Workload Factory aumenta la capacità.



*** *Modalità adattiva*: prevede e modifica la capacità in base ai dati storici.
+
**** *Periodo di analisi*: imposta il periodo di analisi in ore per determinare quanti dati storici considerare per le previsioni di capacità. Il valore predefinito è 120 ore.
**** *Tempo di buffer pianificato*: imposta il tempo di buffer in ore per garantire che la capacità venga aumentata prima che sia effettivamente necessaria. Il valore predefinito è 48 ore.






. Selezionare *Applica*.


.Risultato
Ogni 30 minuti viene eseguita una scansione del file system per determinare se è necessaria capacità aggiuntiva.



== Abilita le notifiche di capacità

Imposta le notifiche per ricevere avvisi quando il file system raggiunge una certa soglia di capacità.

.Prima di iniziare
Devi link:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["abilitare il servizio di notifica Workload Factory"^] per utilizzare questa funzione.

.Fasi
. Accedere utilizzando uno dei link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["esperienze di console"^].
. Seleziona il menu image:workloads-menu-icon.png["L'icona del menu hamburger viene utilizzata per navigare tra carichi di lavoro quali storage, eda, ai, database, vmware e amministrazione."] e quindi seleziona *Archiviazione*.
. Dal menu Archiviazione, selezionare *FSx per ONTAP*.
. Da *FSx for ONTAP*, selezionare il menu delle azioni del file system per abilitare la gestione automatica della capacità.
. Selezionare *Gestisci*.
. In Informazioni, seleziona l'icona a forma di matita accanto a *capacity management*. L'icona a forma di matita appare accanto alla freccia a discesa quando passi il mouse sulla riga *capacity management*.
. Nella finestra di dialogo *gestione della capacità*, seleziona *Abilita notifiche capacità* per attivare le notifiche quando la capacità raggiunge la soglia e ogni volta che NetApp Workload Factory esegue un aumento automatico della capacità.
. Imposta la *Soglia di notifica* per ricevere notifiche quando il file system raggiunge una determinata percentuale di capacità. Il valore predefinito è 70%.
. Selezionare *Applica*.




== Abilita la gestione degli inode per un file system

Abilita la gestione automatica degli inode per aumentare il numero di inode (file) per volume fino al limite consentito.

Ogni 30 minuti viene eseguita una scansione del file system FSx for ONTAP. Verifica gli inode di volume disponibili e controlla se è necessario aggiungere inode incrementali. Il conteggio aumenta in base alle soglie di gestione automatica degli inode.


NOTE: *Utenti Terraform*: Terraform presenta una limitazione che richiede che tutte le operazioni vengano completate all'interno di Terraform.  La gestione degli inode non è supportata in Terraform, ma è possibile abilitare la gestione automatica degli inode nella console Workload Factory.

.Prima di iniziare
Prima di iniziare, prendere in considerazione quanto segue:

* Per gestire automaticamente gli inode del volume, è necessario associare un collegamento. link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["Scopri come associare un collegamento esistente o come creare e associare un nuovo collegamento"]. Dopo aver associato il collegamento, tornare a questa operazione.
* La gestione automatica degli inode può essere impostata con una _soglia di avviso_ che attiva una notifica dal servizio di notifica Workload Factory.  Per utilizzare questa funzione, è necessariolink:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["abilitare il servizio di notifica Workload Factory"^] Primo.
* Devilink:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["concedere credenziali con la politica di autorizzazione _visualizzazione, pianificazione e analisi_"^] in Workload Factory per completare questa attività.


.Fasi
. Accedere utilizzando uno dei link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["esperienze di console"^].
. Seleziona il menu image:workloads-menu-icon.png["L'icona del menu hamburger viene utilizzata per navigare tra carichi di lavoro quali storage, eda, ai, database, vmware e amministrazione."] e quindi seleziona *Archiviazione*.
. Dal menu Archiviazione, selezionare *FSx per ONTAP*.
. Da *FSx for ONTAP*, selezionare il menu delle azioni del file system per abilitare la gestione automatica degli inode.
. Selezionare *Gestisci*.
. In Informazioni, seleziona l'icona della matita accanto a *Gestione automatica degli inode*.  L'icona della matita appare accanto alla freccia a discesa quando il mouse passa sopra la riga *Gestione automatica degli inode*.
. Nella finestra di dialogo *Gestione automatica degli inode*, fornire quanto segue:
+
.. *Credenziali*: Selezionare le credenziali con autorizzazioni _automatizza_ dal menu a discesa.
.. Selezionare il pulsante di abilitazione per *Abilitare la gestione automatica degli inode*.
+
In alternativa, disattivare la funzione.  Se è necessario aumentare il numero di inode, è necessario prima disabilitare la gestione automatica degli inode.

.. *Soglia di avviso*: imposta la soglia di avviso su un valore inferiore all'aumento della soglia per attivare una notifica dal servizio di notifica Workload Factory.  Il valore predefinito è 70%.
+
L'impostazione della soglia di avviso è disponibile solo selink:https://docs.netapp.com/us-en/workload-setup-admin/configure-notifications.html["abilitato il servizio di notifica Workload Factory"^] .

.. *Aumento soglia*: immettere la percentuale massima di aumento per il numero di inode (file) per volume.  Il valore predefinito è 80%.
.. *Aumento incrementale*: immettere la percentuale per aumentare in modo incrementale il numero di inode (file).  Il valore predefinito è il 10%.


. Selezionare *Applica*.


.Risultato
Ogni 30 minuti viene eseguita una scansione del file system per determinare se i volumi necessitano di inode (file) aggiuntivi per volume.
